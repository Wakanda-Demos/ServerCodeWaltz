This will send a request to the server and will call a class method Coop.generateCoopsWithWorker The method will spawn a worker to generate the data. You can generate a million records if you want. Keep in mind that each Coop will have 10 chickens or so - so 1,000,000 will have about 10,000,000 chickens
This will send a request to the server to run class method Coop.generateCoops And set up a progress bar to start polling the server to check on the status of a progress indicator The class method will create the "number of coops" in a loop, updating the pgress indicator as it goes Once it is finished the server will respond with to the initial request and the client call back will be called hiding the pgress bar. If too many coops are sent the client might give up - or the server might error out because it takes to long..... hence a need for a worker>>
